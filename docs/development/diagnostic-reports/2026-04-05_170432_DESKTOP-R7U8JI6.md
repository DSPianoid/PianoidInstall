## Fix Applied

**Root cause:** ASIO driver name hardcoded to "ASIO 2.0 - ESI GIGAPORT eX" (not connected). `ASIOAudioDriver::init()` ignored failed return value from `asioDriver.Init()`, causing ASIO callbacks to never fire, `LockFreeCircularBuffer::produce()` to deadlock, and the engine to stall.

**Fix:** Dynamic ASIO driver enumeration — tries all registered drivers until one initializes. Init return value checked and propagated as exception. Mic device list uses actual loaded driver name.

**ASIO enumeration sequence on this machine:**
1. "ASIO 2.0 - ESI GIGAPORT eX" -- FAIL (not connected)
2. "Ableton Push" -- FAIL (not connected)
3. "Realtek ASIO" -- FAIL (min buffer 1024, need 64)
4. "UMC ASIO Driver" -- SUCCESS (BEHRINGER UMC 1820)

**Note:** ASIO driver enumeration takes ~5-8 seconds (trying 3 failed drivers before finding UMC). The engine needs this time to complete initialization before events can be processed.
