## ASIO Investigation

ASIO Callback (driver type 4) was requested but failed with:
```
RuntimeError: ASIO driver initialization failed - no working ASIO device found
```

**Investigation findings:**
- The ASIO fix commit `f0c5276` (dynamic driver selection, correct output channel mapping) IS incorporated in the current build
- `pianoidCuda.isDriverAvailable(ASIO_CALLBACK)` returns `True` but this only checks **compile-time flags**, not actual hardware availability
- ASIO driver cache file (`asio_driver_cache.txt`) contains `UMC ASIO Driver`
- `UMCAudioCplApp.exe` (Behringer UMC Control Panel) is running
- The C++ code correctly enumerates the UMC ASIO Driver from registry and attempts initialization
- `ASIOInit()` fails at the SDK level — the driver loads but the device cannot be opened
- **Root cause:** The UMC ASIO device is either not powered on, not connected, or has a driver-level issue preventing `ASIOInit()` from succeeding

**Recommendation:** Verify UMC 1820 hardware is powered on and connected via USB. Consider adding a Python-exposed `enumerateAsioDrivers()` function to help diagnose ASIO issues without needing a full engine start.
