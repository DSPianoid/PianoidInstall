### Step 4c: Fix — settings migration for pre-Wave2 localStorage — 11:30
- Problem discovered during /test-ui: existing localStorage entries for `feedInSettings` etc. have no `visualization` key. The old `loadSetting` did `setter(parsed)` which replaces the new default object entirely — so users with saved settings would bypass the `visualization: 'bar'` default.
- Fix: changed `loadSetting` in useSettings.js to merge stored values on top of current defaults via `setter((prev) => ({ ...prev, ...parsed }))`. Guarantees new fields added to a settings object are present even for users whose localStorage predates the field.
- Verified via UI: localStorage started with `{pianoHeight: 30, modesWidth: 20}` (no visualization). After reload, `feedInSettings.visualization = "bar"` appeared in the Settings gear dropdown. ✓
