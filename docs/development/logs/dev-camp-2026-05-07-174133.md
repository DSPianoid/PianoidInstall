## Investigation findings (Step 1)

1. **Complex per-channel mode amplitudes**: stored as `ModeDetection.mode_shape`
   — complex ndarray `(n_channels,)`. After `EspritRunner.chains_to_dicts`,
   serialised as `det["mode_shape"] = [[real, imag], ...]` (list of pairs).
2. **No per-(chain, scenario) complex SCALAR is stored** — only the per-channel
   complex vector. The natural derivation: project the per-channel shape onto
   the chain's reference shape via `np.dot(shape[mask], np.conj(ref))`, where
   `ref` is the unit-normalised mean of all the chain's detection shapes
   (matches the projection in `extract_signed_shapes_per_chain`, which already
   does this — but casts to real with `np.real(...)`). Keeping the imaginary
   part gives a single complex scalar per (chain, scenario).
3. **Scenario indexing**: per user clarification — use simple positional
   integers `0..N-1` matching the canonical sorted-scenarios list (the same
   list `stitched_results["names"]` uses). The folder-derived scenario index
   stays accessible via `stitched_results["names"][k]`.
4. **Chain ordering**: the same one already used by `build_export_payload`
   — `selected_chain_ids` filter, then sort by `frequency_mean` ascending.

→ No PAUSE needed. Data is available; design is clear.

## Implementation summary (Steps 4-7)

* **`extract_complex_amplitudes_per_chain`** — new top-level function in
  `external_export.py`. Mirrors `extract_signed_shapes_per_chain`'s response-
  channel masking and reference-shape construction, but returns the raw
  complex inner product (not its real part) and skips the per-chain
  max-abs normalisation.
* **`write_complex_amplitudes_csv`** — writes the matrix with header row
  `0_re, 0_im, 1_re, 1_im, ..., (N-1)_re, (N-1)_im` (no `mode_idx` column,
  matching the headerless-row convention of the existing `.txt` siblings).
  Missing detections write `0.000000, 0.000000`. `%.6f` formatting.
* **`build_export_payload`** — extended to compute and return
  `complex_amplitudes_per_chain` (length = `effective_num_modes`, NOT padded
  with placeholder rows) and `scenario_names` (list of int).
* **`write_export_files`** — extended to emit `mode_amplitudes.csv`. Guarded
  by `"complex_amplitudes_per_chain" in payload` so older callers that
  build payloads via the old shape continue to work (silently skips the
  CSV instead of crashing).
