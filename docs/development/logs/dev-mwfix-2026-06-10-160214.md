### Step 1: Item 1 root-cause analysis — 2026-06-10T16:12:00Z
- avg-SC aggregate render path: SoundChannelsPane (showAgg) → SoundChannelsAggregateChart → DrawableChart (curve) + a ruler strip below (ModesRule on strings axis, VirtualPiano on modes axis).
- Under H2 (SoundChannelsPane.jsx:121) only the STRINGS axis is reachable for SC → ruler = ModesRule.
- Working reference is RowEditor.js (matrix path): it passes the SAME windowed `range` to BOTH BarChart (startZoom/endZoom) AND ModesRule (`range={range}`) → ruler and chart show identical bucket window → aligned.
- BUG in SoundChannelsAggregateChart.jsx:387: ModesRule gets `range={[0, totalModes-1]}` (ALWAYS the full extent), while the chart's series is SLICED to the visible window via `windowSeries(viewRange=rangeOfModes)` (lines 163-166). Consequences:
  - **Zoomed:** ruler renders all modes [0..totalModes-1]; chart renders only [lo..hi] → gross misalignment (ruler key width = canvasW/totalModes ≠ chart band width = gridW/N_visible).
  - **Full-range:** the ruler key count = `totalModes` (prop) but the chart bucket count = `fullSeries.yValues.length` = length of `aggregateMatrix.averaged`. If those differ (aggregate row length ≠ preset totalModes) the full-range ruler is ALSO misaligned. Even when equal, the FlatBarAxis used in the matrix path would be the consistent choice; ModesRule full-range works only when counts match exactly.
- ROOT CAUSE: the ruler is not derived from the SAME visible bucket set the chart draws. ECharts category x-axis (boundaryGap default true) centers buckets in bands; ModesRule centers keys in `canvasW/totalKeys` slots — they align iff `totalKeys == N_visible` AND the index range matches.
