- ROOT CAUSE (high-confidence, code+measurement): SC matrix canvas onSelectRect fires BOTH onSelectModesRange AND onSelectPitchesRange. SC's onSelectPitchesRange = global setSelectedPitches → a SC drag writes a CHANNEL range (e.g. [1,2]) into the SHARED selectedPitches. The shared zoom-to-selection button (PianoidTuner) then does setRangeOfPitches(selectedPitches) = setRangeOfPitches([1,2]) — poisoning the SHARED rangeOfPitches with a channel range. Feedin/Feedback consume rangeOfPitches with PIANO rows: normRangeStart = pianoRange[0]-firstAvailableNote = 1-23 = -22 → matrix[-22] undefined → matrix[-22][row] → crash.
- So the "missing zoom-IN button" the user saw is actually: button appears, click → crash → app blanks → button (and everything) gone. The real defect is SC's channel-axis selection polluting the shared pitch selection/range, which the shared zoom button propagates to the piano-axis panes.
- FIX (axis-correct, minimal): in SC, do NOT route the channel-ROW-axis selection to the global setSelectedPitches (channel idx must not enter piano-pitch state — same principle as dev-snmtxleak-7e3d selectedChannel decouple). Keep ONLY the mode-axis selection (onSelectModesRange → setSelectedModes) wired. Then the shared zoom button's pitch component sees selectedPitches=null (for SC's own drags) and resets pitch range to full (harmless), while mode zoom works. Also keep SC's selectedPianoRange=null (already so).
