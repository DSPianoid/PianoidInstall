## FIX DESIGN — 2026-06-20T16:19:00Z
- The accent feature must stay (dynamic, user-editable color per pane). The break = the `display:contents` wrapper
  div inserted as the tile's child. The CSS var `--wb-accent` only needs to reach the `.mosaic-window-toolbar`
  (a descendant of `.mosaic-window`). MosaicWindow forwards `className` to `.mosaic-window` but not `style`.
- OPTION CHOSEN (least invasive, keeps feature, restores geometry): set the `--wb-accent` CSS variable on the
  `.mosaic-tile` element — which IS the geometry root mosaic positions — via a tiny side-effect, OR avoid a wrapper
  entirely by injecting the var through the existing `.mosaic-tile` style. Since renderTile only returns the tile's
  CHILD (mosaic owns the .mosaic-tile), I cannot style .mosaic-tile from here directly.
  → Simplest correct fix that preserves geometry: KEEP a single wrapper element but give it `display:contents`'s
    intent WITHOUT removing the box, by making the wrapper a real flex box that FILLS the tile
    (`height:100%;width:100%;display:flex`), so `.mosaic-window`'s `height:100%` resolves against a full-size parent.
    The CSS var rides on this wrapper (inherited into .mosaic-window → toolbar). This restores the exact geometry
    the no-wrapper panes have (window fills tile) while still delivering the dynamic var. Verify live.
