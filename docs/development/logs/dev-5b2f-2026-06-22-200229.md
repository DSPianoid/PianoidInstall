**Design decisions beyond the literal spec (recorded for the report):**
- **D1 â€” Non-streamed completions for the multi-turn/tool path.** The one-shot adapter streams (SSE deltas). For multi-turn + tool_calls, the driver issues NON-streamed `/chat/completions` (request `stream:false`) and parses ONE full response object: `tool_calls` arrive as structured objects with an `arguments` JSON string that would otherwise be split across SSE deltas (brittle to reassemble). The spec describes mapping "`choices[0].message.tool_calls`" (the non-streamed message shape), and `parseStreamPayload` already reads `choices[0].message.content`. So T1 adds a small non-streamed request builder + a `tool_calls` parser, and REUSES the rest. (Streaming token deltas for a pure-chat turn could be a later refinement; non-streamed is correct + simplest for the loop and keeps the whole turn atomic.)
- **D2 â€” `runTool` is an injected callback, default = a ref-using stub.** The driver requires a `runTool(call)=>Promise<string>` to execute tools. T1 does NOT build the real choke-point (piece #4). Default `runTool` returns a clean "no tool runner wired" error string fed back as the tool result (so even mis-wired, it degrades, never crashes). Tests inject a scripted `runTool`.
- **D3 â€” The tool manifest (`tools[]`) is an injected option,** not hard-coded â€” T1 is dormant; the real manifest (pieces #2+#4 tool schemas) is supplied at wiring time (piece #3). Empty manifest â‡’ the model simply gets no tools (a plain multi-turn chat), which is exactly T1's "no tools yet" lower bound + the tool path on top.
- **D4 â€” `send()` appends; `runTurn()` runs.** Per the `SessionDriver` contract `send()` returns void. In T1 `send(turn)` appends the user message and a host pump runs the next turn via the exposed `runTurn()` (which yields the turn's events). The exact lifecycle pump (how the host re-consumes events after a send) is decided in pieces #3/#5; T1 exposes a correct, directly-testable per-turn run. `start()` seeds the conversation + runs turn 1.
