### Design (decided before edit)

**Part 1 — no-prompt flag (start-pianoid.bat):**
- Parse `%1`: accept `/auto` (primary) + `--no-prompt` (alias) → set NOPROMPT=1.
- NOPROMPT=1 skips BOTH the "Press any key to start..." `pause` (line 101-102) AND the trailing "Press any key to exit..." `pause` (line 148-149) — the shortcut-launched window must not hang on a keypress after spawning npm.
- Bare `start-pianoid.bat` (terminal) keeps both prompts (current behaviour byte-identical).

**Part 2 — make-shortcut (NEW make-shortcut.ps1 + make-shortcut.bat wrapper):**
- `make-shortcut.ps1`: WScript.Shell COM `CreateShortcut` → Desktop\Pianoid.lnk; TargetPath=<repo>\start-pianoid.bat, Arguments="/auto", WorkingDirectory=<repo>, IconLocation=<repo>\PianoidTunner\public\favicon.ico,0; `.Save()`. Desktop via `[Environment]::GetFolderPath('Desktop')`. Falls back gracefully if favicon missing (no icon, warn).
- `make-shortcut.bat`: thin wrapper → `powershell -NoProfile -ExecutionPolicy Bypass -File "%~dp0make-shortcut.ps1"` (repo .bat-first convention; user can double-click).

**Part 3 — update-check (NEW check-updates.ps1, called from start-pianoid.bat):**
- Fragile logic concentrated in PowerShell (testable, robust timeout/try-catch), batch stays simple.
- `check-updates.ps1`: for PianoidCore/PianoidTunner/PianoidBasic (current branch) + outer PianoidInstall (master): `git fetch` with a SHORT timeout (Start-Job + Wait-Job -Timeout, kill on overrun) → compare `git rev-list --count HEAD..@{u}` (>0 = origin ahead). If git missing / fetch fails / no upstream / no network → treat that repo as "unknown", never error. If ANY repo ahead → `[System.Windows.Forms.MessageBox]::Show(...)` Yes/No. Yes → exit 10 (=> batch calls update-repos.bat). No → exit 0. Nothing ahead / all-unreachable → exit 0. ANY unhandled failure → exit 0 (fall through to launch). Entire script body in try/catch; the .bat call is itself wrapped so a non-zero/crash still proceeds to launch.
- start-pianoid.bat: after dir checks, before launch — `powershell -NoProfile -ExecutionPolicy Bypass -File check-updates.ps1`; if errorlevel==10 → call update-repos.bat (then continue to launch). Any other code → just launch. Wrapped so it NEVER blocks/hangs/errors the launch.
