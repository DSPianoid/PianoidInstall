### CHECKPOINT — DERIVED BOUND (KEY FINDING; contradicts the spec's assumed numerator)

Von-Neumann substitution u^n_p = g^n e^{iθp}, θ=kΔx∈[0,π]. Spatial operators →
2cosθ (2nd-order), 2cos2θ (4th-order), d3 → −2(1−cosθ). Characteristic quadratic:
  g² − A(θ)·g − B0(θ) = 0
  A = shift_0 + 2·shift_1·cosθ + 2·shift_2·cos2θ + cfd·(−2)(1−cosθ)
  B0 = shift_b + cfd·2(1−cosθ)
With dec_curr=cfd=0: B0=+1 → g²−Ag+1=0 → product of roots =1 → stable ⟺ |A|≤2 ∀θ.

At NYQUIST θ=π (the binding mode): A(π)=2+32B−4T. |A(π)|≤2 gives the EXACT two-sided box:
  8·coeff_bending ≤ coeff_tension ≤ 1 + 8·coeff_bending           (T,B units from Kernels.cu)
- UPPER edge (the CFL limit): coeff_tension − 8·coeff_bending ≤ 1.   ← CFL_LIMIT = 1.
- LOWER edge: coeff_tension ≥ 8·coeff_bending (tension must dominate bending at grid scale).

NUMERIC CONFIRMATION (dev-cfl-upper-boundary, exact amplification factor):
  the invariant (coeff_tension − 8·coeff_bending) = 1.000000 to 6 digits along the
  whole upper boundary for B∈[0,0.1] (drifts only at B≥0.15, far outside real presets).
  vs (T+4B)<=1: 298/551 grid-cell MISMATCHES with true stability — NOT a valid criterion.
  Concrete: T=0.39,B=0.05 → T+4B=0.59 (passes ratio) but |g|=1.22 DIVERGES (false-negative);
            T=0.85,B=0.10 → T+4B=1.25 (fails ratio) but |g|=1.0 STABLE (false-positive).

DAMPING EFFECTS (measured): velocity damping dec_curr does NOT change the bound
  (T_upper=1.08 for all dec∈[0,2] at B=0.01 — A and B0 both scale by dec_inv, |g|≤1 invariant).
  HF damping cfd TIGHTENS it (cfd=0.1 → T_upper 1.08→0.88) — so ignoring cfd is CONSERVATIVE.

REAL-PRESET REGIME (Belarus_8band_196modes, 88 pitches, authoritative Pitch.get_coefficients):
  coeff_tension ∈ [0, 0.046] (≪1, ~20× upper-CFL margin); coeff_bending ∈ [−0.0047, 0] (NEGATIVE
  in this preset — jung stored negative). All 88 stable. So in practice the engine sits far from
  the upper CFL edge; the length→dx regression failed the OPPOSITE way (dx ~84–196× too LARGE →
  T,B→0 → g²−2g+1=0 DOUBLE root at |g|=1, defective → polynomial drift = the "noise that persists").

IMPLICATIONS for the spec:
  1. The user-fixed numerator (coeff_tension + 4·coeff_bending) is NOT the scheme's stability
     criterion. Correct invariant: (coeff_tension − 8·coeff_bending), CFL_LIMIT=1 (sign of bending
     term is MINUS, coeff is 8 not 4). This is exactly the high-stakes fact the proposal flagged.
  2. A single ratio ≤ 1 cannot capture stability fully — there are TWO edges (upper CFL + lower
     bending-Nyquist) PLUS the degenerate T,B→0 drift (caught only by isfinite + the lower edge
     when B is tiny). The catastrophic length→dx family is the T,B→0 end, caught by isfinite.
  → REPORTING to team-lead before implementing the guard (numerator contradicts measured scheme).
