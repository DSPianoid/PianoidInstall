The ctx-distortion reset+flush+clear sequence is irrelevant to this silence — it correctly zeroes mode state, the event queue re-excites at cycle 0, and the kernel produces audio that ends up in `recorded_audio_`. It just doesn't end up in `rawSoundBuffer`.
