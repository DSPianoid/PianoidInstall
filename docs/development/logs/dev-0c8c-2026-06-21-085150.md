### Step 1 finding — the two race windows
- (W1) deadline-too-tight: a legit turn >60s on 1M Opus is murdered. Fix: config + default 180s.

**Design note (interval vs deadline default):** raising the deadline default to 180s while keeping the interval default at 120s makes interval < deadline. The original inline comment ("120s > the 60s deadline") relied on interval > deadline so a fresh idle ping never armed while a prior ping's deadline was live. With interval(120s) < deadline(180s) a second scheduler tick COULD fire while a prior ping's deadline is still armed — but pingLiveness is idle-skipped (a turn in flight = no-op) and `clearPingTimer()` is called at the top of every armed ping (so the second ping REPLACES the first's deadline rather than racing it), and the callback re-validation (W2 fix) prevents a stale fire. So interval<deadline is safe. The interval DEFAULT stays 120s per the brief, and the index.ts comment is corrected.
