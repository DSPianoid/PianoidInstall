### Step 4: Design + Implement — 2026-06-09T19:03:00Z

**P1 Authority:** the .bat orchestrates the pre-launch sequence (sole owner of launch-flow control). The two new .ps1 helpers are pure queries/decisions returning an exit code — they own NO persistent state; the running-servers helper performs a port-scoped kill ONLY when it has shown the pop-up and the user clicked Kill (the helper is the sole actor for that kill, mirroring how check-updates.ps1 owns the update decision). No non-owner writes any shared state. Compliant.

**P2 Concern:** each .ps1 = one concern (running-servers check / CUDA check), matching the existing one-concern-per-helper pattern (check-updates.ps1 = update check). The .bat gains two more pre-launch gate invocations — same concern it already has (orchestrate prereqs/update/launch). No concern bleed.

**Exit-code contracts (mirroring check-updates.ps1's 10/0):**
- check-running-servers.ps1: `20` = servers were up AND user chose Cancel (→ .bat aborts launch); `0` = proceed (no servers up, OR user chose Kill&restart [kill already done in-helper], OR any best-effort failure).
- check-cuda.ps1: `30` = user clicked Cancel on a warning pop-up (no device / SM<60) (→ .bat aborts launch); `0` = proceed (device present & SM≥60 silently, OR user clicked Continue on a warning, OR best-effort can't-determine → skip silently).

**FLAG INTERACTION decision:** running-servers + CUDA pop-ups are SAFETY/CORRECTNESS prompts (not the cosmetic "press any key"), so per the brief they SHOULD still show under /auto. BUT a desktop-shortcut (/auto) launch is unattended — a modal pop-up that blocks forever with nobody to click it would hang the shortcut. RESOLUTION (flagging as a design question to team-lead): under /auto, the helpers run in a NON-INTERACTIVE mode — they still DETECT and PRINT to the console, but instead of a blocking MessageBox they apply a safe default and continue: running-servers → DO NOT kill (leave the running stack alone, just warn on console — killing the user's stack unattended is the dangerous default), launch proceeds; CUDA warn → proceed (informational). This keeps /auto truly unattended while bare/interactive runs get the full pop-up. Passing an `-Auto` switch to the .ps1 from the .bat when NOPROMPT=1.
