**Measurement #3 тАФ Connection state vs. real dispatch:**

**Web MIDI spec note:** Per W3C Web MIDI API spec section 6, setting `onmidimessage` on a `MIDIInput` SHOULD implicitly call `open()` ("Implementations are encouraged to make a reasonable attempt to open the input when onmidimessage is set"). Chromium implementations have varied historically тАФ older versions auto-opened, newer versions do not always.

With `connection: "closed"`, hardware MIDI keypresses never reach `handleMIDIMessage` тЖТ `playNote` тЖТ `midiPlayNote` (the Fix-MIDI wrapper).
