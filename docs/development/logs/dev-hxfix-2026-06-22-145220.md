ROOT CAUSE: loudness is PRESET-DEPENDENT (mode count, feedin/feedback coupling, deck differ per preset). A single global excitation constant `c` tuned for ONE preset cannot keep all presets in range; BaselinePreset1 is ~30 dB hotter than the Belarus anchor and clips.
RECOMMENDED APPROACH (for user/orchestrator decision — HOLD): an OUTPUT-STAGE normalization/limiter so ANY preset stays in range, rather than per-preset excitation re-calibration. Two sub-options:

  (B) A preset-load output AUTO-NORMALIZE pass: measure a reference note's peak at load and scale main_volume_coefficient (or a new output_gain) to land the preset at a target headroom — keeps dynamics linear, preset-agnostic, but needs a measurement/normalize step at load (cf. parked dev-d52b headroom/limiter design + project_reset_output_loop_regression unscaled-output).
