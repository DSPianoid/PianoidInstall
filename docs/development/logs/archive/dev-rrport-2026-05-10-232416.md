### Issue 1 — Hidden 7th RR module: `calibration_validator_v2.py` (749 LOC)

The proposal lists 6 files for porting (~3700 LOC). However, `scenario_averager.py` (already in PianoidCore) lazy-imports a 7th RR module that is NOT in the port list:

```
PianoidCore/pianoid_middleware/modal_adapter/scenario_averager.py:781,1410:
    from calibration_validator_v2 import (CalibrationValidatorV2, QualityThresholds)
```

The same module is also imported by 3 existing tests:
- `tests/integration/test_scenario_averager.py:51,619`
- `tests/integration/test_modal_create_from_zip.py:548,792`
- `tests/unit/test_qc_curves.py:46`

**If we delete `_room_response_bootstrap.py` without porting `calibration_validator_v2.py`**, `scenario_averager.ensure_averaged_responses` and these tests will fail to find it on `sys.path`.

**Decision needed:**
- (A) **Port calibration_validator_v2 too** as a 7th file, ~750 LOC, into `measurement/calibration_validator.py`. Update scenario_averager + 3 test files to use the new path. This is faithful to "self-contained" Phase 0 goal, costs +1 day.
- (B) **Defer it** — keep `_room_response_bootstrap.py` alive but reduced to ONLY discovering `calibration_validator_v2`. Phase 0 is incomplete-but-shippable; Phase 0.5 ports the validator. Less faithful to Gate 1.
- (C) Port now into `pianoid_middleware/modal_adapter/` directly (NOT under `measurement/`) since it logically belongs to averaging not collection. Same LOC, different home.

**Recommendation:** (A). Phase 0 goal is "self-contained PianoidCore"; leaving one RR module dangling defeats it.

### Issue 2 — `recorderConfig.json` lookup at `collection_engine.py:534-545`

`MeasurementSession._load_recorder_config()` reads `recorderConfig.json` from the **RR repo root** as the "default" config, falling back to a built-in dict when missing. Today this is the Belarus-tuned configuration (multichannel_config, calibration_quality_config, truncate_config) committed in `D:/repos/RoomResponse/recorderConfig.json`.

After the port, the RR repo location may not exist or may diverge. We need to decide:
- (A) **Vendor the RR-shipped recorderConfig.json** into `PianoidCore/pianoid_middleware/modal_adapter/measurement/default_recorderConfig.json` and drop the RR-repo lookup. Default config lives in-tree.
- (B) **Drop the RR lookup, keep the built-in dict default only.** Loses the multichannel_config defaults.
- (C) **Keep the lookup behind the env var `PIANOID_ROOMRESPONSE_PATH`** as an opt-in escape hatch (kills the cleanness of Phase 0 but preserves "everything works as before" for power users).

**Recommendation:** (A). Phase 0 promise is "last 'everything works as before' snapshot"; vendoring the shipped config is the cleanest equivalent.

### Issue 3 — `belarus_reextract.py` tool uses `bootstrap_roomresponse()` directly

`PianoidCore/tools/grid_search/belarus_reextract.py:19-37` imports and calls `bootstrap_roomresponse()` to ensure RR is on sys.path before running averaging. After deletion of the shim, this 60-LOC tool needs a 4-line edit (drop the bootstrap call entirely — the in-tree imports will work directly).

**Recommendation:** Simple edit, do as part of Phase 0. No PAUSE needed; flagged so locks include this file.

### Issue 4 — Build script integration: which `detect_paths.py` and which `build_config.json` for sdl_audio_core?

RR's sdl_audio_core has its OWN `detect_paths.py` (319 LOC) and produces its OWN `build_config.json`. PianoidCore has the **canonical** `detect_paths.py` (`PianoidInstall/detect_paths.py`, 848 LOC; `PianoidCore/detect_paths.py`, 507 LOC) which already produces `pianoid_cuda/build_config.json` with the exact keys sdl_audio_core's setup.py reads (`include_dirs`, `library_dirs`, `libraries`, `sdl3_root`).

Two options:
- (A) **Reuse the existing `pianoid_cuda/build_config.json`** — sdl_audio_core's setup.py reads from the same path. Simplest. Risk: pianoid_cuda's config has `cuda_arch_list` etc that sdl_audio_core ignores, no conflict.
- (B) **Have build_pianoid_cuda.bat write a separate `sdl_audio_core/build_config.json`** with a tailored subset.

**Recommendation:** (A). Same JSON, no duplication. The build script change is simply: after pianoidCuda install completes, run `pip install --no-cache-dir --force-reinstall --no-deps PianoidCore/sdl_audio_core/` (~30 LOC of bat script). sdl_audio_core's setup.py reads `Path(__file__).parent / "build_config.json"` — we'd either copy `pianoid_cuda/build_config.json` to `sdl_audio_core/build_config.json` before pip install, or pass `PIANOID_BUILD_CONFIG` env var (sdl_audio_core's setup.py would need a tiny patch to honour it — diverging from RR by ~5 LOC).

**Recommended micro-decision within (A):** copy the build_config.json — keeps sdl_audio_core's setup.py byte-identical to RR's, no source divergence.
