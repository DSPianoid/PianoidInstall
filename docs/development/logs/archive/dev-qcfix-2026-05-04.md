# dev-qcfix — QC Visualization "no curves available" regression fix

**Started:** 2026-05-04 (orchestrator-spawned)
**Task:** Fix QCVisualizationPanel showing "No QC curves available" for every (scenario, channel) selection.

## User report (verbatim)

> "I get No QC curves available for Scenario 11 channel 2. The scenario may
> lack raw_recordings or the channel may have too few cycles for split-half
> analysis. - whatever I choose"

User selects ANY scenario × channel pair in the QC Visualization Panel and
gets the same generic error. Suspected regression from dev-cd0c's regex fix.

## Investigation

1. **Listed actual raw files** for `PlyWoodTake1_4/PlyWood-Scenario11-Take1/raw_recordings/`
   and `PlyWoodTake1_1/...` — files are named `raw_PlyWood-Scenario11-Take1_NNN_<ts>_chN.npy`,
   matching dev-cd0c's tightened regex `^raw_.*_ch(\d+)\.npy$`.
2. **Ran `_list_raw_files_per_channel` directly** against both projects in a Python
   one-liner with the canonical venv: returned 5 files per channel ch0–ch7
   on both. **Regex is NOT the regression.**
3. **Ran `pool_scenario_cycles` + `compute_qc_curves_for_channel(ch=2)`** against
   both: produced n_cycles_total=19 and t_eff_ms=26.1 — fully working.
4. **Hit the live HTTP endpoint** for several (project, scenario, channel) combos
   on the running modal-adapter at 127.0.0.1:5001 — all returned HTTP 200 with
   valid payloads.
5. **Hit the endpoint with `scenario=Scenario%2011`** (the way the FRONTEND sends
   it when `scenarioInfo.names` is missing): returned HTTP 404 with
   `"Scenario 'Scenario 11' not found under project ..."`.

## Root cause

`QCVisualizationPanel.jsx` `onScenarioChange` falls back to the menu LABEL
when `scenarioInfo.names` is not provided (see line 535:
`setScenarioName(opt?.label || \`Scenario${idx}\`)`). That sends `"Scenario 11"`
(with a space) to the backend, which expects the folder basename
`"PlyWood-Scenario11-Take1"`. The backend correctly returns 404, the frontend's
`fetchQCCurves` catch block returns `null`, and the panel renders the
misleading "No QC curves available... too few cycles for split-half" message.

`/modal/scenario_info` (the endpoint that populates `scenarioInfo`) returns
`{loaded, processed, total_loaded, total_processed, unprocessed}` — it never
provided `names`. Bug has been there since dev-3151 landed the panel, the
regression report misattributed it to dev-cd0c (whose regex fix is correct).

## Fix

Backend-only minimal fix: have `scenario_info()` also return a `names` map
`{idx: folder_basename}` for the current project, derived from
`_scenario_folders_for_project(current_project_path)` + `_extract_scenario_index`.
The frontend `QCVisualizationPanel` already prefers `scenarioInfo.names[idx]`
when present (line 135), so no frontend change required.
