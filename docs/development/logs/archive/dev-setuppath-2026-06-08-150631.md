### Step 4: Edit Code — 2026-06-08T15:10:00Z
[PROGRESS] 2026-06-08T15:10:00Z step=4 note=design + locks + DMC

**P1 (Authority):** State touched = persistent Machine/User Windows PATH. Normally owned by the OS installers. My change makes `setup-dev.ps1` a *restorative* writer — it snapshots the pre-existing PATH and re-adds only entries the installers DROPPED. The script already orchestrates those installers, so reconciling the damage they cause is within its authority; it invents no new PATH semantics, only preserves what was there.

**P2 (Concern):** `setup-dev.ps1`'s single concern = "set up the PianoidCore dev env by installing toolchain components." PATH preservation is a defensive sub-concern of that same job. To keep the concern cleanly separated WITHIN the file-set, the pure snapshot/diff/restore/truncation logic goes into a SEPARATE dot-sourced module `setup-path-guard.ps1` (concern = "PATH snapshot/diff/restore math, no side effects on env"); `setup-dev.ps1` calls it. This also solves the load-time-execution problem: `setup-dev.ps1` runs `main` at load, so a test cannot dot-source IT without triggering Assert-Admin + the whole install. Dot-sourcing the side-effect-free guard module is safe → unit-testable in isolation.

**Routing (Step 4b):** PowerShell installer logic with Windows-environment semantics + judgment about PS 5.1 idioms is NOT DeepSeek-eligible (DeepSeek = Python/JS/TS with fast isolated test gate). Written inline by Opus. No /fn spawn (single self-contained file-set, judgment work, below the ~$0.15 spawn tax).

**Design:**
- NEW `setup-path-guard.ps1` (pure helpers, side-effect-free except the explicitly-named restore/backup fns):
  - `Get-PersistentPathSnapshot` — reads Machine + User PATH via `[Environment]::GetEnvironmentVariable('PATH', scope)`; returns an ordered, split, trimmed, de-duplicated list per scope (object with `.Machine` / `.User` arrays + raw strings).
  - `Get-DroppedPathEntries -Before <string[]> -After <string[]>` — pure set-diff: every entry present in Before but absent from After (case-insensitive compare, the Windows PATH convention), order = Before order. Empty-safe.
  - `Merge-RestoredPath -CurrentEntries <string[]> -DroppedEntries <string[]>` — append dropped entries to the surviving current list, de-duplicated (case-insensitive), preserving the order of survivors then the dropped entries; returns the joined `;` string. PURE (no env write).
  - `Test-PathWithinLimit -PathString <string>` — returns $true if length ≤ a safe cap (`PATH_LENGTH_SAFE_CAP` = 2047, the classic per-variable ceiling for legacy apps; expansion of REG_EXPAND_SZ caps at 32767 but 2047 is the safe floor). Pure.
  - `Find-NationalInstrumentsPaths` — detection: checks `C:\Program Files (x86)\National Instruments`, `C:\Program Files\National Instruments`, any `*CVI*` child dir, or an NI/CVI entry already on the (passed-in) PATH; returns the list of evidence found. Pure-ish (filesystem reads only, no mutation).
  - `Write-PathBackup -Machine <string> -User <string> -Directory <string>` — writes a timestamped `pianoid-setup-path-backup-<ts>.txt` with both raw strings for manual recovery; returns the file path. (Named side effect.)
  - `Restore-DroppedPathEntries -Snapshot <obj> -BackupDirectory <string>` — the orchestration wrapper: re-reads current persistent PATH, diffs vs snapshot per scope, and for each scope with drops: if the merged value passes Test-PathWithinLimit → `[Environment]::SetEnvironmentVariable('PATH', merged, scope)` and report; else → WARN loudly, do NOT write, keep the backup. Returns a summary object. (The ONLY env-writing fn.)
- EDIT `setup-dev.ps1`:
  - Dot-source `setup-path-guard.ps1` near the top (after param/`$ErrorActionPreference`).
  - In `main`, BEFORE `Assert-Admin`'s installers run (right after Assert-Admin, before Ensure-Python): snapshot persistent PATH, write the timestamped backup, run NI/CVI detection + print a clear heads-up.
  - Python installer arg `PrependPath=1` → config-driven, default `0` (script already returns python.exe's explicit `.Source`, so the rest of the run does not need Python prepended to the persistent PATH).
  - At END of main, after all installers + before/around Write-BuildConfig: call `Restore-DroppedPathEntries` to re-append anything dropped.
