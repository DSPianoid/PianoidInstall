# dev-qccmb — QC Combined view: signals + diff (not ratio)

**Started:** 2026-05-04 (orchestrator-spawned)
**Task:** Fix Combined view mode in `QCVisualizationPanel.jsx` to render
signal_a + signal_b + difference (not ratio). The ratio belongs only in
the Ratio view mode.

## User report (verbatim)

> "Combined should show two curves and the difference, not ratio."

## Investigation

Current Combined view (line 480ff):
- Dual y-axis (left = amplitude, right = ratio).
- Three series: signal_a, signal_b, ratio.
- Right axis colored teal `#80cbc4` (matching the ratio).

Difference view (the source of truth for diff styling):
- Single y-axis, name "amplitude".
- Series colors: `signal_a − signal_b (raw diff)` = `#ce93d8` (light
  purple), `env_diff (smoothed)` = `#f48fb1`.

Curves view convention:
- signal_a (half A) = `#4fc3f7` (light blue).
- signal_b (half B) = `#ffb74d` (amber).
- These are the same colors used in the existing Combined view's
  signal_a / signal_b series — preserve them.

## Plan

Rewrite the Combined return-block:
- Keep dual y-axis, but change the right axis from "ratio" (0..0.5) to
  "diff" (auto-range, color matched to the diff series purple).
- Replace the `ratio` series with `signal_a − signal_b` (raw diff) on
  the right y-axis. Color `#ce93d8` to match the Difference view.
- Keep signal_a (`#4fc3f7`) and signal_b (`#ffb74d`) on the left axis.
- Update the legend strings.

Why dual axis (not single): signal_a/signal_b can be O(1.0) damped
sinusoids; diff = a − b is typically much smaller (the whole point of
the QC split-half check is that a ≈ b in ideal case). Putting both on
one axis would either dwarf the diff (single axis at signal scale) or
clip the signals (axis at diff scale). Dual axis lets each be visible.

## Constraints

- Frontend-only. No backend.
- Tests must stay green (229 baseline + 14 in QCVisualizationPanel).
- The user's "use theme.palette" guidance is reasonable but the entire
  file is hex-based today (signal_a/signal_b/diff colors all hex). A
  theme refactor would touch every series across all 4 view modes —
  out of scope for this UX patch. Reuse the existing hex constants that
  are already established for signal_a / signal_b / diff so visual
  continuity is preserved when the user toggles between view modes.
  Note as future work in the commit.
