### Step 4 — Boundary mode discovery, 19:25
- First implementation used `mode="nearest"` for `gaussian_filter1d`. Live verification on `PlyWoodTake11/Sc14/ch5` returned `t_eff = 999.98 ms` — wildly late. Diagnosis: Hilbert magnitude of a finite signal has a boundary build-up artefact at the last samples (analytic-signal FFT treats input as periodic; discontinuity drives `|H[N-1]|` upward). With `mode="nearest"` the inflated boundary value gets replicated as padding for the Gaussian and overwhelms the smoothed envelope at the end.
- Switched to `mode="reflect"` — mirrors the decaying tail across the boundary, preserving the local trend. Same scenario now reports 207 ms. Documented in the docstring + MODAL_ADAPTER_GUIDE.
- Added regression test `test_decayed_signal_with_noisy_tail_does_not_extend_to_end` to lock this in.
