### Diagnosis (root cause, with confidence: high)

**Stale frontend state leaks across project switches.** Sequence:

1. User opens PlyWoodTake1_3 (line layout). Tracking runs / loads.
   `chainEditor.editedChains` and `stabilizationData` are populated
   from PlyWoodTake1_3's chains. `chainEditor.editedChains[12]` =
   freq 116.83, 25 dets, stable.
2. User switches to PlyWoodTake1_4 (grid layout). `useModalAdapter.openProject`
   calls `setStages(INITIAL_STAGES)` which clears `tracking.data`,
   then `syncFromBackend()` re-fetches `/modal/tracking_results` so
   `stages.tracking.data.chains` updates to PlyWoodTake1_4's chains.
   - **BUT** `setSelectedChains([])` is NOT called.
   - **AND** `setStabilizationData(null)` is NOT called.
   - **AND** `chainEditor.discardEdits()` / `chainEditor.initFromServer(...)`
     is NOT called (the existing useEffect gates re-init on
     `editedChains === null`).
3. `ModalResultsView` line 587 picks up the stale `chainEditor.editedChains`
   (still PlyWoodTake1_3's chains) for the stab-diagram render.
4. The stab-diagram renders chain_id=12 with 25 stable detections at
   PlyWoodTake1_3's scenarios.
5. User clicks chain 12 â†’ `selectedChains = [12]`.
6. `GridHeatmapInset` is rendered (because `layoutType === "grid"`
   from PlyWoodTake1_4). It calls `getGridHeatmap(12)` which hits
   `/modal/grid_heatmap/12`. The backend uses
   `_tracked_chains[12]` from PlyWoodTake1_4 â†’ returns 6 cells with
   amplitude (24 cells with `null` because PlyWoodTake1_4's chain 12
   doesn't have detections at those scenarios).
7. User sees: chart says chain 12 has 25 points, heatmap shows 6.

The two endpoints (`/tracking_results` indirectly via stale
`chainEditor.editedChains` and `/grid_heatmap/<id>`) are reading from
**different projects**.

### Side issue (also a stale-state bug, lower priority)

After `chainEditor.saveEditedChains` returns (re-indexed by backend),
`chainEditor.editedChains` retains old IDs until `syncFromBackend`
runs and the React.useEffect at line 200 fires. But that effect
doesn't fire because `editedChains !== null`. So the local chain
editor state never re-syncs after a save â†’ subsequent clicks on
"chain N" (frontend) call `getGridHeatmap(N_old)` which the backend
maps to a now-different chain. This is a separate manifestation of
the same root cause class: state ownership unclear, no project- or
operation-boundary reset.
