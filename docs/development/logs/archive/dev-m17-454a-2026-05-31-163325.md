[EDIT] file=PianoidTunner/src/components/newWindowChart.jsx
[FILE-LOC] PianoidTunner/src/components/newWindowChart.jsx before=552 after=572

1. **Removed `return api.abort`** â€” the `fetchedRef` fence already guarantees only one POST fires, so there is nothing to abort. Letting the POST complete naturally is the right behaviour now.
2. **Added `isMountedRef = useRef(true)`** + guard on every `setState` inside `.then`/`.catch`. If the component truly unmounts (real navigation-away, not StrictMode's faux-unmount), the response handlers short-circuit instead of leaking late setState onto an unmounted root.

Without this reset, the second effect would early-return with `isMountedRef` still false, and the POST's `.then` would skip setState â€” same Loading-hang in different clothes.
