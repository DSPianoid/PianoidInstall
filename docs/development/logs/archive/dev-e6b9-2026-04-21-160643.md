## Root Cause & Fix Summary

**Root cause:** `/play_keyboard` offline mode calls `_stop_online_engine(pianoid)` before rendering. That deactivates the audio driver. A parallel mic-capture call made during offline rendering has no audio being pushed to the speakers, so the mic records only ambient room noise (peak ~100/32767 int16, RMS ~2.9, 1% non-zero). There is no existing endpoint to play an arbitrary WAV through the driver while the mic records.

**Fix:** Added `capture_mic` (bool, default false) + `tail_ms` (float, default 2000, max 10000) to `/play_keyboard` online mode. When enabled, the handler starts mic capture, schedules the sweep through the live audio driver (speakers audible), sleeps synchronously for `num_pitches * speed_ms_per_note + tail_ms`, stops capture, and writes an un-normalized 16-bit PCM WAV to `D:/tmp/keyboard_mic_<timestamp>.wav`. Offline mode is unchanged — it still produces the peak-normalized synth WAV as before.

**Verification:** New mic WAV peak 0.121 (≈3960 int16), RMS 0.0154 (≈503 int16), 99.95% non-zero. Previous run: peak 100, RMS 2.9, 1% non-zero. All three thresholds in the team-lead brief exceeded by a wide margin.
