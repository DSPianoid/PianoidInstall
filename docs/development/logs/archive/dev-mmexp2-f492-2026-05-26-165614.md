**Bug analysis (from user spec + code trace):**

1. The export-set filter for `omega_coef.txt` lives in `external_export.build_export_payload()` lines 539-543:
   ```python
   if selected_chain_ids is None:
       active_chains = list(chains)
   else:
       sel = set(int(x) for x in selected_chain_ids)
       active_chains = [c for c in chains if int(c.get("chain_id", -1)) in sel]
   ```
   `active_chains` then drives every export array (omega_coef.txt, Q_coeff_*.txt, decka_coeff.txt, Ci_coef_cos.txt, mode_amplitudes.csv).

2. The new `write_relative_modal_mass_txt()` (lines 783-878) iterates `modal_mass_summary["chains"]` directly. It is called from `write_export_files()` (line 941-949), which receives NO `selected_chain_ids` parameter. Result: all 757 summary chains land in the file (vs ~12 in omega_coef.txt for a typical export set).

3. Data flow: UI `selectedChains` → POST body `selected_chains` (project_routes.py:583) → `adapter.export_to_text_files(selected_chain_ids=sel)` (project_routes.py:593-595) → `export_text_files(..., selected_chain_ids=sel)` → `build_export_payload(selected_chain_ids=sel)` (filter applied) BUT `write_export_files(...)` receives no selected_chain_ids.

**Fix plan:**

- Plumb `selected_chain_ids` through `export_text_files()` → `write_export_files()` → `write_relative_modal_mass_txt()` so the latter can apply the same filter.
- Apply identical filter in writer: `if selected_chain_ids is None: chains stay as-is; else: filter by chain_id in set`.
- Defensive: also drop rows where both `m_relative` and `m_absolute` are non-finite (NaN/None) after the filter — covers the case where a chain in the export set has no modal-mass data.
- Update header comment with "# filter: chains in export set (matches omega_coef.txt)" so readers know row count won't necessarily equal `chain_count`.
