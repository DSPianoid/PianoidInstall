#### P1 (Separation of Authority)
State touched: the input caret position (`selectionStart`/`selectionEnd` on the DOM `<input>`).
Owner: NumInput itself owns the caret-restore decision; the DOM input owns the live caret.
Currently THREE writers race for it: `useLayoutEffect`, `preserveCursorPosition` (setTimeout),
`scheduleCursorRestore`'s rAF. That is a P1 violation — three writers, no single owner.
The fix RESTORES P1: `useLayoutEffect` becomes the SOLE writer of the post-render caret;
`pendingCursorRef` is the single piece of state it reads. Net: 3 writers → 1.

#### P2 (Separation of Concern)
Module edited: NumInput.js — its concern is "a numeric input editor". Caret preservation
across re-renders is squarely within that concern. The fix does not widen the module; it
removes redundant mechanisms. No concern bleed.
