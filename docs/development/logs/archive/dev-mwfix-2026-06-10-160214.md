- BUG in SoundChannelsAggregateChart.jsx:387: ModesRule gets `range={[0, totalModes-1]}` (ALWAYS the full extent), while the chart's series is SLICED to the visible window via `windowSeries(viewRange=rangeOfModes)` (lines 163-166). Consequences:
  - **Zoomed:** ruler renders all modes [0..totalModes-1]; chart renders only [lo..hi] → gross misalignment (ruler key width = canvasW/totalModes ≠ chart band width = gridW/N_visible).

If those differ (aggregate row length ≠ preset totalModes) the full-range ruler is ALSO misaligned. Even when equal, the FlatBarAxis used in the matrix path would be the consistent choice; ModesRule full-range works only when counts match exactly.
- ROOT CAUSE: the ruler is not derived from the SAME visible bucket set the chart draws. ECharts category x-axis (boundaryGap default true) centers buckets in bands; ModesRule centers keys in `canvasW/totalKeys` slots — they align iff `totalKeys == N_visible` AND the index range matches.

FEATURE 2 (apply-function, ANCHORED) — the design crux:
- UI (in the WB toolbar, MUI v6 dark): ONE "Apply function" IconButton + a shape <Select>
  (dropdown) + a NumInput showing the driving coefficient `k`; the button ALSO carries a
  non-passive wheel handler (useAxisWheel-style) that nudges `k` (up=+step, down=−step) and
  re-applies live so the user sees the curve morph as they wheel.
- Shapes: linear, quadratic, cubic, exponential, logarithmic, power, sigmoid (S-curve).
- ★ANCHOR MATH (curve passes through the current cell, value UNCHANGED):
    Let the selected index range be [lo..hi], anchor index a = the current cell's index
    (clamped into [lo..hi]); anchor value V = vector[a] (the existing value — the fixed point).
    Define a normalized signed distance from the anchor for each index i in [lo..hi]:
        d(i) = (i - a) / span,  where span = max(a-lo, hi-a) or 1 (so d ∈ [-1, +1], d(a)=0).
    Define a shape function s(d) with s(0)=0 (so the anchor is unmoved) and s monotonic in |d|:
        linear:      s = d
        quadratic:   s = sign(d)*d^2
