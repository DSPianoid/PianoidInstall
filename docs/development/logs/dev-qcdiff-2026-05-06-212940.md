### Investigation — 23:00

**Bug 1.** Follow-up #1's audit concluded the dispatch was Y-only because `yAxisIndex: [0, 1]` was on the payload. User reported the X-axis effect persists. Re-examined via context7 ECharts docs:

> The `dataZoom` action signature is:
> ```ts
> dispatchAction({
>   type: 'dataZoom',
>   dataZoomIndex: number,    // index of dataZoom COMPONENT
>   start, end, startValue, endValue,
> })
> ```
> No `xAxisIndex`/`yAxisIndex` parameters. Without `dataZoomIndex` the action fans out to ALL dataZoom components.

ROOT CAUSE: `baseDataZoom` has 2 components (Y `inside` at index 0, X `slider` at index 1). The previous dispatch with `yAxisIndex: [0, 1]` (an unrecognised property) was silently ignored — start/end then applied to BOTH components, including the X slider, moving the visible time range as a side-effect.

Live-verified in chrome-devtools by injecting ECharts 5.5.0 directly:
- buggy form (`yAxisIndex:[0,1]`, start:25, end:75): Y dataZoom → [25,75], X dataZoom → [25,75] (X SLIDER MOVED)
- fixed form (`dataZoomIndex:0`, startValue:-0.5, endValue:0.5): Y dataZoom → [25,75], X dataZoom → [0,100] (X UNCHANGED)

Fix: dispatch with `dataZoomIndex: 0` + `startValue`/`endValue` (data-space).

**Bug 2.** Previous geometry was `(start+end)/2 = midpoint preserved`, which drifts when the user previously panned. New rule (per user spec):
- zero-bottom views (Envelopes, Ratio): startValue=0 fixed; endValue scales by factor
- zero-center views (Curves, Difference, Combined): startValue=-newHalf, endValue=+newHalf

Anchor declared per VIEW_MODES entry as `yZoomAnchor`. Full data extent computed from curves payload (per-view: max of relevant series). Current visible window read from chart's dataZoom state.

**Bug 3.** Live-curl of new v3 schema on PlyWoodTake11/Sc0/ch1 (after dev-qcthr's modal-adapter restart):
- `smoothing_kind` field added; `smoothing_ms` = 20.0 (was 5.0)
- env_signal end-tail: trailing-100 max / mid-signal mean = **1.04×** (was 1.11× on v1)
- env_signal[-1] = 0.000687 (decays smoothly, no spike)
- Resolution: deferred-resolved by dev-qcthr's Phase 1. Dropped from this session's scope.

**Bug 4.** "All-combined view: combined + envelopes, zero-based". Ambiguous — literal interpretation clips negative half of signed signals (~-0.19 floor on a typical IR). Per orchestrator instruction, surfacing for clarification before implementing.
