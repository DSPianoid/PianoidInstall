## DESIGN DECISION — C scope: SINGLE per-preset global normalization factor
The volume feeds loudness LINEARLY and uniformly (temporal_impulse linear in vol). A SINGLE factor s = 1/global_max applied to EVERY pitch's levels_matrix[:,2,:] preserves ALL relative relationships (cross-pitch balance, cross-level dynamics, cross-component shape) and lands every volume in [0:1] with global max -> 1.0. Per-pitch normalization would BREAK cross-pitch balance (REJECTED). The absolute loudness drop from s is exactly what B's per-preset scale compensates. This is the clean 3-layer separation: curve=[0:1] relative shape, A=energy physics, B=per-preset absolute scale.
- Hook (load + legacy migration): StringMap.normalize_excitation_volumes() called once after pitches load in __init__. Idempotent: if global_max already <= 1.0 (+eps) it's a no-op (already normalized). preserve-shape = divide-by-global-max.
- Save: writes the normalized [0:1] values automatically (Pitch.pack_for_saving_preset packs levels_matrix, now normalized).
- B2 retire: with [0:1] volumes the ~1e8 magnitude is gone; acoustic_tuner factor-clamp comment reconciled (no functional clamp change needed — it clamps the dimensionless correction factor, not the absolute coefficient).

## ★B ARCHITECTURE RISK (volume_center overwrite) — under investigation
Brief says "fold scale into volume_center." But volume_center is treated GLOBAL: the FE recomputes it as exp((presetVolume+64)/8) and pushes via set_runtime_parameters; switch_preset snapshots/restores it. If the per-preset scale is folded into volume_center, the FE's next push OVERWRITES it -> scale lost. Need a separate multiplicative output_scale that survives FE volume_center pushes WITHOUT a CUDA field. Investigating the FE push cadence + the cleanest pure-Python seam.

## ★B DESIGN RESOLVED — per-preset `output_scale` multiplier on volume_center at every set-site
FE computes volume_center LOCALLY = exp((presetVolume+64)/8) and pushes via set_runtime_parameters (BottomBar onVolumeSensitivityChange); it does NOT read the returned value back into its compute. So folding the per-preset scale into the SERVER-SIDE volume_center, applied at EVERY site that sets volume_center, survives FE overwrites without a CUDA field or FE edit:
- New per-preset field `mp.output_scale` (ModelParams.py: __init__ + PARAM_NAMES + pack(); default 1.0; legacy presets -> 1.0). Cached in preset model_parameters. Persists on /save_preset for free.
- Single chokepoint helper pianoid._effective_volume_center(requested) = requested * float(mp.output_scale). Applied at: init_pianoid:2304 seed, switch_preset:3084 restore, and the set_runtime_parameters helper (backendServer.py:243-244). => the engine's effective coefficient at slider 64 = requested_center * output_scale; FE pushes its own center, server layers the per-preset scale on top.
- output_scale COMPUTATION (at preset load): render p60 v127 offline (audio_off) at the CURRENT mvc, measure RAW post-volume sint peak via load_sint_audio_from_kernel(mvc); scale = TARGET(1.706e9) / peak; but since peak already includes the CURRENT volume_center (which includes any old output_scale), compute against the BARE coefficient. Cleaner: render gives pre-volume soundFloat peak; output_scale = TARGET / (soundFloat_peak * base_volume_center_at_slider64). base_center = exp((presetVolume+64)/8) with presetVolume=100 default. So output_scale = 1.706e9 / (soundFloat_peak * exp(20.5)). CACHE in mp.output_scale: compute on first load if absent (==1.0 default sentinel means "not yet calibrated" -> compute); reuse if present (!=1.0 OR an explicit calibrated flag); re-measure if energy params changed (defer: recompute when absent only, v1).
  NOTE: distinguish "legacy/uncalibrated (1.0 default)" from "calibrated to 1.0". Use a separate persisted bool `output_scale_calibrated` (default False) to avoid re-calibrating a deliberately-1.0 preset and to trigger first-load calibration. compute iff not output_scale_calibrated.
