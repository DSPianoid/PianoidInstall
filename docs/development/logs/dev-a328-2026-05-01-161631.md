### Step 5/7: Verification (live UI via /pianoid-ui) — 17:55 onward

12/12 verification checkpoints passed **after two bugs were caught and fixed mid-verification**:

**Bug 1: PresetPanel library list collapses to 0 height in small panes (clickability bug)**
- Root cause: PresetPanel's outer Box uses `display: flex; flexDirection: column`. The Library Box had `flex: 1` but the surrounding sections (Current Preset, footer Apply, dividers) had no `flexShrink: 0`. In a small pane (~280px tall), the surrounding sections consumed all space, leaving the `flex: 1` Library Box at height 0. List items rendered in DOM but pointer-events hit the parent Box, causing `chrome-devtools click` to time out.
- Fix: added `flexShrink: 0` to Section 1 wrapper, Section 2 header, both dividers, and footer Apply Box; raised Library Box `minHeight` from 0 to 80px.

**Bug 2: Layout migration walker creates duplicate IDs (Mosaic crash)**
- Root cause: when the saved layout already contained both a deprecated leaf (Settings) AND the target leaf (Preset) — which happens when a user manually re-adds the Settings pane via the Window Layout Manager (verified in checkpoint 11), then F5 reloads — the walker rewrote Settings → Preset, producing two `"Preset"` leaves. Mosaic does not support duplicate IDs and crashes hard with `Duplicate IDs [Preset] detected`.
- Fix: rewrote the walker as a two-pass operation. First pass collects all leaf IDs in a Set via `collectLeafIds`. Second pass via `mapDeprecatedPaneIdsWith(node, existingIds)` does the rewrite, but if the rewrite target ID already exists in the tree, drops the deprecated leaf (returns `null`) instead of creating a conflict. The user's existing target pane is preserved; the deprecated leaf is silently dropped.

Both fixes are in the Phase 2 commit (caught + fixed before commit).
