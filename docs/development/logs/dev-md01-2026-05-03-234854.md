- **Confirmed root cause** ‚Äî the dedup state is split into two independent stores: `_ws_last_command` (per-SID, WS only) and `last_command` (module-global, REST only). When a duplicate crosses transports OR comes from a different WS SID, neither store catches it. The pre-WebSocket-era single filter (`last_command` only, REST only) caught everything because REST was the only ingress.
