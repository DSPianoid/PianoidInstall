### Step 1: Root cause (source) — 2026-06-16T14:14:00Z
SAVE-BY-REFERENCE aliasing at 3 boundaries — every saved config aliases the ONE live layout-tree object, which react-mosaic mutates in place:
- mosaicConfigStore.saveConfigAs:121 → `{ ...configs, [name]: liveLayout }` stores liveLayout BY REFERENCE (no clone).
- PianoidTuner live-mirror effect (~1276-1283) → `{ ...prev, [active]: layout }` writes the live `layout` BY REFERENCE into the active config every layout change.
- PianoidTuner handleMosaicConfigSelect:1070 → `setLayout(mosaicConfigs[name])` loads the stored object BY REFERENCE → subsequent edits mutate the stored snapshot.
⇒ Save A, reconfigure, Save B: all entries point at the same mutating object → "both names show the last config" (user's bug).
FIX (team-lead spec): deep-copy (snapshot) at save-as, at the mirror write, and at select/apply load. Add a cloneLayout(structuredClone w/ JSON fallback) helper in the store. Layout tree is plain JSON (strings + {first,second,direction,splitPercentage}) → structuredClone/JSON-roundtrip safe.
