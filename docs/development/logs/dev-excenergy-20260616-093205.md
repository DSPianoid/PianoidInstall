## ★ WAVE 2 BLOCKER (measured) — per-note slot is INT, coefficient is fractional float

DesIGN GAP found by measurement (not inferred): the per-note volume slot the proposal's
confirmation (a) designated as the coefficient carrier is **integer-typed end to end**:
- `gaussTest.cuh:20` kernel sig: `int* string_excitation_params`
- `Pianoid.cuh:131` host: `std::vector<int> string_excitation_params`
- `Pianoid.cu:65` sized `MAX_STRINGS_PER_EVENT*3` (int); device `dev_string_excitation_params`
  registered as an int buffer; `Pianoid_excitation.cu:136` memcpy as int.
The original commented-out `real volume = string_excitation_params[blockIdx.x*3+1]` was an
int→real read — i.e. the legacy volume_coefficient in this slot was an INTEGER. But the B2
coefficient `c·m·v·tImpulse·sImpulse` is a FRACTIONAL float (typically << 1 to ~tens).
Stored in an int slot it TRUNCATES → silent loudness corruption (often →0). The locked
proposal (confirmation a) assumed the slot "just works" as a per-(pitch,velocity) carrier;
it did not address the int ABI. **This is a high-stakes ABI/data-model decision — NOT to be
inferred (per the high-stakes-inference rule).**

Resolution options (need team-lead/user decision):
- **A. Separate real buffer** — add `real* dev_string_excitation_coeff` (host vector<real>,
  device real buffer, NEW kernel arg to gaussKernel). Clean types; but adds ONE kernel arg
  (contradicts the proposal's "no new kernel arg" — minor, still no device reshape of the
  existing buffers). Most correct.
- **B. Fixed-point in the int slot** — store round(coeff*SCALE) as int, divide by SCALE in
  the kernel. No ABI change, but lossy + a magic SCALE constant; fragile for a wide dynamic
  range (ppp→fff spans ~18×, and c calibration can be small).
