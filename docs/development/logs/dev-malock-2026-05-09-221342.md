### Root cause

`ModalAdapter.jsx` line 930:

```jsx
<MappingEditor
  ...
  disabled={settingsFrozen || layoutType === "grid"}
/>
```

The `layoutType === "grid"` clause permanently disables Channel Mapping for any grid-layout project, regardless of ESPRIT state. This violates the documented lock-after-ESPRIT contract.

**History.** This expression originates from commit `4b6acfb` (`dev-b9dd`, 2026-05-04) where the Channel Mapping accordion was nested inside a `{measurementInfo && layoutType === "grid" && (...)}` block. In that context the `layoutType === "grid"` clause was redundant but harmless — the accordion only rendered for grid layout in the first place.

Commit `6ddb615` (`dev-8b5f`, 2026-05-04) refactored the Project subpanel and widened the render gate from `measurementInfo && layoutType === "grid"` to just `measurementInfo &&` — so the Channel Mapping accordion now renders for BOTH line and grid layouts. **But the `disabled={settingsFrozen || layoutType === "grid"}` prop was carried through unchanged.** That stale clause is what locks the settings on grid-layout fresh projects.

The Band Configuration accordion (line 974, `disabled={settingsFrozen}`) was correctly updated in the same dev-8b5f refactor; only the MappingEditor clause was missed. This is why the user sees Band Config unlocked but Channel Mapping locked on a fresh grid project.
