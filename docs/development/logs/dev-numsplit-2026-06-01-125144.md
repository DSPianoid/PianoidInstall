### Split design (R-1 aligned, behavior+API preserving)
Approach: extract the duplicated/pure logic into a co-located pure-JS helper module `numInputMath.js`. These are the biggest, safest, most-duplicated units (review R-1 named the exponent-step collapse as "the single biggest reduction ~250 LOC"). Pure functions = trivially behavior-preserving + directly unit-testable.

`NumInput/numInputMath.js` (NEW, pure JS, no React, no JSX):
  - `generateUniqueId()` — moved verbatim (NumInput.js:15-16).
  - `formatNumber(num, exp, places)` — moved verbatim (the forceExp + toFixed/toExponential logic, NumInput.js:112-129).
  - `anchorExponentCaret(oldDisplay, newDisplay, oldCaret)` — moved verbatim (NumInput.js:251-280).
  - `getStepFromCursorPosition(displayValue, cursorPosition, fallbackStep)` — moved from NumInput.js:439-588; the ONLY change is parameterizing the 3 closure reads (inputRef.current.selectionStart → cursorPosition arg; displayValue → arg; internalStep → fallbackStep arg). Body logic byte-identical.
  - `computeExponentStep({ displayValue, cursorPosition, direction, minValue, maxValue, decPlaces })` → returns `{ appliedDisplayValue, committedValue, newCaret }` OR null if cursor not in exponent part. Collapses the EXACT shared body of the 4 duplicated blocks (split mantissa/exp → posFromEnd → powerForDigit → newExp → rebuild display+sign → Number → clamp → branch constrained-vs-raw → anchorExponentCaret). The component keeps the 3 side-effects (setDisplayValue/handleValueChange/scheduleCursorRestore) + the in-exponent-part guard. Differs across the 4 sites ONLY by `direction` (wheel: deltaY>0?-1:1; ArrowUp/Down: key; up-btn: +1; down-btn: -1) → passed as arg.

NumInput.js keeps: all React state, all event handlers, the JSX, handleValueChange (the integer-round + clamp + onChange choke point stays IN the component — it closes over onChange/minValue/maxValue/integer/displayValue/isExponential), updateDisplayValue, the caret refs + useLayoutEffect, getStepValue (thin wrapper calling the pure getStepFromCursorPosition), getLiveNumericValue.
