# dev-de0f: Fix CurveEditor chart not updating on RCM capture

**Started:** 2026-04-18 16:19
**Task:** RCM captures reference points (counter shows "Capturing mf - 4 refs") but the CurveEditor ECharts chart doesn't re-render to show new reference points.

## Status: Fixed and verified in browser

## Root Cause

The original session identified three infrastructure issues (RCM status sync, unstable onEvents, merge mode). Those fixes were necessary but not sufficient.

The **actual visual problem**: reference points were rendered as per-data-item `itemStyle` overrides on a `line` series (yellow color, larger `symbolSize`). With 88 data points on a ~400px chart and all coefficients at 1.0, the yellow dots were visually indistinguishable from the red line -- same Y position, occluded by neighboring points and the line itself.

## Fix (session 2)

**Replaced per-item styling with a dedicated scatter series.** Reference points are now rendered as a separate `scatter` series (type `"scatter"`, symbol `"diamond"`, symbolSize 14, z-index 10) overlaid on top of the line series. The scatter series uses `"-"` for non-reference positions (hidden by ECharts). This guarantees reference markers are always visually prominent regardless of chart density or coefficient values.
