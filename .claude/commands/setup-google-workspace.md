# Google Workspace MCP Setup

Set up the Google Workspace MCP server for Gmail and Google Calendar access.

## Prerequisites
- Python 3.10+ installed
- `uv` package manager installed (or pip)

## Steps

### 1. Install workspace-mcp
```bash
uv tool install workspace-mcp
```
This installs `workspace-mcp.exe` to `~/.local/bin/`. If `uv` is not at the default path, find it:
```bash
where uv.exe
```

### 2. Get user's Gmail address
If `$ARGUMENTS` contains an email, use it. Otherwise ask for their Gmail address.

### 3. Create Google Cloud OAuth credentials
Guide the user through these browser steps:

1. Go to [console.cloud.google.com](https://console.cloud.google.com/)
2. Create a new project (e.g. `claude-mcp`) or select existing
3. **APIs & Services → Library** → Enable:
   - **Gmail API**
   - **Google Calendar API**
4. **APIs & Services → OAuth consent screen**:
   - Choose **External** → Create
   - Fill in App name, support email, developer contact email
   - Click through remaining steps
5. **Add test user**: In OAuth consent screen → Audience/Test Users → add the user's Gmail address
6. **APIs & Services → Credentials → Create Credentials → OAuth client ID**:
   - Application type: **Desktop app**
   - Name: e.g. `Claude Code`
7. Copy **Client ID** and **Client Secret** from the created credential

### 4. Add to `~/.claude.json`
Read `~/.claude.json`, find or create the `mcpServers` object, and add:
```json
"google-workspace": {
  "command": "C:\\Users\\astri\\.local\\bin\\workspace-mcp.exe",
  "args": ["--tools", "gmail", "calendar", "--single-user"],
  "env": {
    "GOOGLE_OAUTH_CLIENT_ID": "<client-id>",
    "GOOGLE_OAUTH_CLIENT_SECRET": "<client-secret>",
    "OAUTHLIB_INSECURE_TRANSPORT": "1"
  }
}
```

### 5. Run OAuth authentication
This is the critical step. The auth flow requires a running HTTP server to capture the OAuth callback.

**Start the server in background:**
```bash
export GOOGLE_OAUTH_CLIENT_ID="<client-id>"
export GOOGLE_OAUTH_CLIENT_SECRET="<client-secret>"
export OAUTHLIB_INSECURE_TRANSPORT=1
~/.local/bin/workspace-mcp.exe --tools gmail calendar --single-user --transport streamable-http
```
Run in background (run_in_background=true).

**Trigger auth via MCP protocol** (this ensures the state token matches the running server):
```bash
SESSION_ID=$(curl -s -D - -X POST http://localhost:8000/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"setup","version":"1.0"}}}' \
  2>&1 | grep -i "mcp-session-id" | tr -d '\r' | awk '{print $2}')

curl -s -X POST http://localhost:8000/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -H "Mcp-Session-Id: $SESSION_ID" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"start_google_auth","arguments":{"service_name":"gmail","user_google_email":"<user-email>"}}}'
```

**Extract the Authorization URL** from the response and present it to the user as a clickable link.

Tell the user:
1. Click the authorization link
2. Sign in with their Google account
3. They'll see "Google hasn't verified this app" — click **Continue** (or Advanced → Go to app)
4. Grant all requested permissions
5. Confirm when done

**After success**, verify credentials were saved:
```bash
ls ~/.google_workspace_mcp/credentials/
```
Should contain `<email>.json`.

Kill the temp server:
```bash
taskkill //F //IM workspace-mcp.exe
```

### 6. Reload VS Code
Tell the user: `Ctrl+Shift+P` → "Developer: Reload Window", then open a new chat.

### 7. Verify
Test with a simple tool call:
- `search_gmail_messages` with query `in:inbox`
- `list_calendars`

## Important Notes
- **Do NOT use CLI mode** (`--cli start_google_auth`) to generate the auth URL — the state token won't match the callback server. Always use the MCP protocol method above.
- OAuth consent screen must be in **test mode** with the user's email added as a test user, OR the app must be published/verified.
- Credentials are stored in `~/.google_workspace_mcp/credentials/<email>.json`
- Token refresh is automatic; re-auth only needed if scopes change or token is revoked.
- To add more Google services later, add to the `--tools` list (e.g., `drive`, `docs`, `sheets`).

## Troubleshooting
- **"Access blocked: app not verified"**: Add user's email as test user in OAuth consent screen → Audience
- **"Invalid or expired OAuth state"**: The auth URL was generated by a different process than the callback server. Restart the server and re-generate the URL using the MCP protocol method.
- **Port 8000 in use**: Kill old processes: `taskkill //F //IM workspace-mcp.exe`
- **MCP tools not appearing**: Reload VS Code and start a new chat.
